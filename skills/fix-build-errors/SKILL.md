# Fix Build Errors

## Workflow

### Step 1: Collect All Errors

```bash
npm run build 2>&1
```

If only type-checking (no Vite build):

```bash
npx tsc --noEmit 2>&1
```

Capture the **full** error output. Do NOT stop at the first error.

### Step 2: Categorize Errors

Group errors into these categories before fixing. Fix them in this order (dependencies first):

1. **Missing modules (TS2307)** â€” wrong import paths or missing packages
2. **Unused variables/imports (TS6133)** â€” dead code
3. **Type incompatibilities (TS2322, TS2345, TS2769, TS2352, TS2304)** â€” type mismatches

### Step 3: Batch Fix by Category

Use parallel subagents or batch StrReplace calls. Each category has its own fix pattern:

#### Category 1: Missing Modules (TS2307)

Common causes in this project:
- **`react-router-dom`** â†’ React Router v7 merges everything into `react-router`
- **`@cloudflare/workers-types`** â†’ Removed; types auto-generated by `npm run cf-typegen` into `worker-configuration.d.ts`

Fix: Use `StrReplace` with `replace_all=true` per file to update import paths.

#### Category 2: Unused Variables (TS6133)

Two sub-patterns:

| Situation | Fix |
|---|---|
| Unused **import** | Remove from import statement |
| Unused **function parameter** (but part of required signature) | Prefix with `_` (e.g. `env` â†’ `_env`) |

#### Category 3: Type Incompatibilities

Common patterns in this stack:

| Error Pattern | Fix |
|---|---|
| `JWK` not assignable to `JsonWebKey` (jose â†’ WebCrypto) | Cast: `jwk as JsonWebKey` |
| `Promise<T>` not assignable to `T` | Make function `async`, use `await` instead of `.then()` |
| `string \| undefined` not assignable to `string` (Hono params) | Non-null assertion: `c.req.param("id")!` |
| `Env` not assignable to `Record<string, unknown>` | Double cast: `env as unknown as Record<string, unknown>` |
| R2 `httpMetadata` not assignable to `HeadersInit` | Extract specific fields: `cached.httpMetadata?.contentType` |
| Browser API not in types (e.g. `cookieStore`) | Replace with universally typed API (e.g. `document.cookie`) |
| Component prop doesn't exist (e.g. `asChild` on Base UI) | Remove the prop; Base UI uses `render` prop pattern, not `asChild` |

### Step 4: Verify

```bash
npm run build 2>&1
```

Must exit with code 0. If new errors appear, repeat from Step 2.

### Step 5: Update Docs

If any fix reveals a **design-level issue** (wrong package name in docs, wrong API choice, wrong router type), update the corresponding file in `/docs/`.

Check with:

```bash
# Search docs for outdated references
rg "react-router-dom" docs/
rg "createBrowserRouter" docs/
rg "@cloudflare/workers-types" docs/
```

## Project-Specific Notes

- **TypeScript config**: `tsc -b` uses project references (`tsconfig.json` â†’ `tsconfig.worker.json` + `tsconfig.client.json`)
- **Build command**: `tsc -b && vite build` (type-check then bundle)
- **Type generation**: After changing `wrangler.json`, run `npm run cf-typegen` to regenerate `worker-configuration.d.ts`
- **Secrets**: Declared via `declare global { namespace Cloudflare { interface Env { ... } } }` in `src/server/env.ts`
- **UI components**: Based on Base UI (not Radix), do NOT use `asChild` prop â€” use `render` prop or direct children instead
